# RepoRadar 前端设计文档

> 报告页面的组件结构、图表配置、布局细节与视觉规范。
> 日期：2026-06-12

---

## 1. 技术选型

| 技术 | 版本 | 引入方式 |
|---|---|---|
| Vue3 | 3.x | CDN `<script>` |
| ECharts | 5.x | CDN `<script>` |
| TailwindCSS | 3.x | CDN `<script>` |

无构建步骤，无 npm，单 HTML 文件内完成全部渲染。

### 2.2 宽度与间距

- 页面最大宽度：`1280px`，水平居中
- 左右栏间距：`24px`
- 卡片间距：`16px`
- 卡片圆角：`8px`
- 卡片阴影：`shadow-sm`（TailwindCSS）

## 3. 组件设计

### 3.1 Header 组件

**数据源**：`__DATA__.repository` + `__DATA__.meta`

```
┌─────────────────────────────────────────────────────────┐
│  📊 RepoRadar                                           │
│                                                         │
│  my-project / main                                      │
│  分析周期：2024-01-01 ~ 2024-01-31                       │
│                                                         │
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────────────────┐    │
│  │ 1234 │  │  12  │  │ 456  │  │  健康度 72/100   │    │
│  │ 提交 │  │ 作者 │  │ 文件 │  │  ████████░░      │    │
│  └──────┘  └──────┘  └──────┘  └──────────────────┘    │
└─────────────────────────────────────────────────────────┘
```

- 健康度评分用进度条展示，颜色分级：
  - ≥ 80 绿色 (`green-500`)
  - ≥ 50 黄色 (`yellow-500`)
  - < 50 红色 (`red-500`)

### 3.2 团队活跃热力图

**数据源**：`__DATA__.activityHeatmap`

**ECharts 配置**：
- 图表类型：`heatmap`
- 横轴 (x)：周一 ~ 周日
- 纵轴 (y)：0时 ~ 23时
- 颜色映射：提交数从少到多 → 浅蓝 → 深蓝
- Tooltip：显示具体提交数

```
     周一  周二  周三  周四  周五  周六  周日
 0时  ░    ░    ░    ░    ░    ░    ░
 1时  ░    ░    ░    ░    ░    ░    ░
 ...
 9时  ▓    ▓    ▓    ▓    ▓    ░    ░
10时  █    █    █    ▓    ▓    ░    ░
14时  █    █    ▓    █    █    ░    ░
 ...
23时  ░    ░    ░    ░    ░    ░    ░
```

### 3.3 作者贡献排行

**数据源**：`__DATA__.authors`

**ECharts 配置**：
- 图表类型：水平柱状图 (`bar`)
- 双系列：新增行数（绿色） / 删除行数（红色）
- 按 commit 数降序排列，取 top 10
- Y 轴：作者名
- X 轴：代码行数

### 3.4 作者画像卡片

**数据源**：`__DATA__.authors`

每位作者一张卡片，滚动排列：

```
┌─────────────────────────────────────┐
│  Alice                              │
│  alice@example.com                  │
│                                     │
│  提交: 120  新增: +5,000  删除: -2,000  │
│  活跃时段: 09:00 - 15:00            │
│  主力模块: order, payment           │
│                                     │
│  🤖 AI 画像:                        │
│  Alice 是订单和支付模块的核心贡献者，  │
│  提交集中在工作日上午，代码增删比健康， │
│  提交信息规范度较高。                 │
│                                     │
│  [小时分布迷你图 - Sparkline]         │
└─────────────────────────────────────┘
```

- AI 画像区域：`v-if="author.profile"` 控制，降级模式下隐藏
- 迷你图：用 SVG 或 CSS 简单柱状图展示 24 小时分布，不依赖 ECharts

### 3.5 异常总览卡片

**数据源**：`__DATA__.anomalies`

```
┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
│    5     │ │    3     │ │    2     │ │   18    │
│ 巨型提交 │ │易挥发文件│ │跨域提交  │ │模糊提交 │
│  🔴 高   │ │  🟡 中  │ │  🟡 中  │ │  🟠 低  │
└─────────┘ └─────────┘ └─────────┘ └─────────┘
```

- 每类异常一张统计卡，显示数量
- 底部标注严重程度指示色（根据该类异常中最高 riskLevel 着色）

### 3.6 异常详情列表

**数据源**：`__DATA__.anomalies.giantCommits` / `volatileFiles` / `crossDomainCommits` / `vagueCommits`

Tab 切换展示四类异常：

```
┌──────────────────────────────────────────────┐
│  [巨型提交] [易挥发文件] [跨域提交] [模糊提交]  │
├──────────────────────────────────────────────┤
│                                              │
│  abc1234 · Alice · 2024-01-15                │
│  "refactor: rewrite payment flow"            │
│  变动: 823行 · 25文件 · 模块: order, payment  │
│                                              │
│  🤖 AI 分析:                                 │
│  意图: 重写支付流程以解决并发竞态条件           │
│  标签: REFACTOR, BUGFIX                       │
│  风险: 🔴 HIGH                                │
│                                              │
│  ──────────────────────────────────────────  │
│                                              │
│  def5678 · Bob · 2024-01-20                  │
│  ...                                         │
│                                              │
└──────────────────────────────────────────────┘
```

- AI 分析区块：`v-if="commit.analysis"` 控制
- 风险等级颜色：LOW=绿、MEDIUM=黄、HIGH=红、CRITICAL=深红
- Tags 用彩色 badge 展示

## 4. 视觉规范

### 4.1 配色方案

基于深色主题（更适合数据展示场景）：

| 用途 | 色值 | TailwindCSS |
|---|---|---|
| 页面背景 | `#0f172a` | `slate-900` |
| 卡片背景 | `#1e293b` | `slate-800` |
| 主文字 | `#e2e8f0` | `slate-200` |
| 次文字 | `#94a3b8` | `slate-400` |
| 主色调 | `#3b82f6` | `blue-500` |
| 成功/低风险 | `#22c55e` | `green-500` |
| 警告/中风险 | `#eab308` | `yellow-500` |
| 危险/高风险 | `#ef4444` | `red-500` |
| 严重 | `#dc2626` | `red-600` |

### 4.2 字体

- 标题：系统默认 sans-serif（TailwindCSS 默认栈）
- 代码/Hash：`font-mono`（monospace 栈）
- 正文：`text-sm`（14px）

### 4.3 交互

- Tab 切换：点击切换，无动画（MVP）
- 卡片 hover：`hover:shadow-md`，过渡 `transition-shadow duration-200`
- 无弹窗、无模态框
- 无路由（单页面，所有内容在一个滚动视图中）

## 5. 降级渲染规则

| 条件 | 行为 |
|---|---|
| `meta.llmEnabled === false` | Header 显示降级提示 badge；AI 画像/洞察/建议区域全部隐藏；全局洞察只显示健康度 |
| `author.profile === null` | 作者卡片中 AI 画像区块隐藏 |
| `commit.analysis === null` | 异常详情中 AI 分析区块隐藏 |
| `globalInsight.summary === null` | Footer 总结和建议区域隐藏 |
| `module.insight === null` | 风险解读面板显示 "未配置 LLM" 提示 |

所有 `null` 检查均通过 `v-if` / `v-else` 实现，确保页面在降级模式下不显示空白区域。
